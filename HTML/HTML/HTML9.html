    포함하고 있는 태그를 부모태그,
    포함된 태그를 자식 태그라고 부름.

    HTML에 있는 여러 태그들은
    서로 부모 자식 관계가 막 바뀜

    어떤 태그들은
    자식이 있는 곳엔 항상 부모가 있고
    부모가 있는 곳에 항상 특정한 자식 태그가 있는,
    태그들이 몇개 있음.

    


    1. HTML<br>
    2. CSS<br>
    3. JavaScript<br>

    제법 그럴듯한 목차

    But,
    HTML을 만든 사람들은

    그래야 그 태그가 의미에 맞게 사용되는 거라고 할 수 있음.

    이 때 사용하는것.

    띄어쓰기를 하고싶음

    br태그를 쓰면 되겠지만
    이럴때 쓰라고 만든 태그가 있음
    li태그의 부모인
    <ul>
    이라는 태그

    이 항목이 어디서부터 어디까지가 서로 연관된 항목인지를
    경계를 짓기 위한, 그룹핑을 하기 위한 부모 태그가 필요한 것

    <li>태그는 반드시
    부모태그를 가지고 있음.

    <ul>태그는 반드시
    자식태그를 가지고 있다.

    
    

    만약 목차가 1억개라고 생각해보자.
    1, 2, 3이라는 숫자를 1억번 써야 될 것임.

    근데 누가 와서
    " 어 미안해, 여기 첫 줄은 지워줬으면 좋겠어 "
    라고 하면

    밑에 있는 2 목록을 1로 바꿔야 되고
    1억개의 목록을 하나하나 다 바꿔야 될 거야

    이런 경우에 구원해줄수 있는 태그
    <ul> -> <ol>

    앞에 자동으로 숫자가 넘버링 됨

    ol
    Ordered List
    ul
    Unordered List
